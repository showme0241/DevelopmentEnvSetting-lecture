# ESLint
ESLint는 ECMAScript 코드에서 문제점을 검사하고,
일부는 더 나은 코드로 정정하는 린트 도구 중의 하나다.

코드의 가독성을 높이고, 잠재적인 오류와 버그를 제거해 단단한 코드를 만드는 것이 목적이다.
과거 JSLint, JSHint에 이어서 최근에는 ESLint를 많이 사용하는 편이다.

코드에서 검사하는 항목을 크게 분류하면 아래 두 가지다.
- 포맷팅
- 코드 품질

포맷팅 : 일관된 코드 스타일을 유지하도록 하고 개발자로 하여금 쉽게 읽히는 코드를 만들어준다.
    ex) "들여쓰기 규칙", "코드 라인의 최대 너비 규칙"을 따르는 코드가 가독성이 더 좋다.

코드 품질 : 어플리케이션의 잠재적인 오류나 버그를 예방하기 위함이다.
    사용하지 않는 변수 쓰지 않기, 글로벌 스코프 함부로 다루지 않기 등이 오류 발생 확률을 줄여 준다.


## 린트가 필요한 이유
협업을 위해 품질이 보장된 코드를 조건부 규칙으로 관리하기 위함
=> 코드의 규칙을 모두 일관성있게 관리하여 쓸데없는 commit을 방지하고, review 및 read를 위한 가독성 향상

    - 초기 eslint 설정값 세팅
        npx eslint --init => 실행하면 상황에 맞는 Q&A 제공 후, 초기 설정값 업데이트
